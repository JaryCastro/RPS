# Contributing to RPS

We welcome contributions! To ensure a smooth review process, please follow these requirements:

### Coding Standards
* **Style:** Please follow standard Swift naming conventions.
* **Comments:** All new functions should have a brief comment explaining their purpose.

### Requirements for Pull Requests
* **Functionality:** Code must compile without errors.
* **Testing:** If you add a new game logic (like a new move), please include a basic test case.
* **No Breaking Changes:** Ensure your changes do not break the existing game loop.

### Process
1. Fork the repo and create your branch.
2. Commit your changes with clear messages.
3. Submit a Pull Request for review.
